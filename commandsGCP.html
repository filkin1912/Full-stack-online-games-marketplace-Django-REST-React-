Games Store Backend – Full Workflow Summary

1. Backend & Cloudinary + Docker Workflow

Step Action Essential Commands / Changes
1 Install Cloudinary packages pip install cloudinary django-cloudinary-storage
2 Add Cloudinary credentials Add to .env.local:
CLOUDINARY_CLOUD_NAME=... CLOUDINARY_API_KEY=... CLOUDINARY_API_SECRET=...
3 Configure Django for Cloudinary In settings.py:
- Add cloudinary and cloudinary_storage to INSTALLED_APPS
- Set DEFAULT_FILE_STORAGE = "cloudinary_storage.storage.MediaCloudinaryStorage"
- Add CLOUDINARY_STORAGE = {"CLOUD_NAME": ..., ...}
4 Update models to use Cloudinary Replace ImageField with CloudinaryField in your Django models.
5 Apply migrations python manage.py makemigrations
python manage.py migrate
6 Improve production settings In settings.py:
- Load env vars
- Configure ALLOWED_HOSTS, CORS_ALLOWED_ORIGINS, CSRF_TRUSTED_ORIGINS
- Set DEBUG = False for production
- Configure static files (STATIC_ROOT, WhiteNoise)
7 Fix CSRF scheme issue Ensure all CSRF / CORS origins start with http:// or https://.
8 Fix Docker build error In requirements.txt:
- Remove psycopg2
- Keep psycopg2-binary==2.9.9
9 Create production Dockerfile Dockerfile (essentials):
- Base: python:3.12-slim
- Install system deps
- COPY requirements.txt .
- RUN pip install -r requirements.txt
- COPY . .
- RUN python manage.py collectstatic --noinput
- CMD ["gunicorn", "exam_project.wsgi:application", "--bind", "0.0.0.0:8080"]
10 Build new backend image docker build -t filk1912/gamesplay-backend:v2 .
11 Verify Artifact Registry repo Confirm in Google Cloud → Artifact Registry:
- Repository: games-store-repo
- Location: europe-west4
- Format: Docker
12 Tag image for Artifact Registry docker tag filk1912/gamesplay-backend:v2 europe-west4-docker.pkg.dev/games-store-490014/games-store-repo/gamesplay-backend:v2
13 Authenticate Google Cloud gcloud auth login
14 Allow Docker to push to Artifact Registry gcloud auth configure-docker europe-west4-docker.pkg.dev
15 Push image to Artifact Registry docker push europe-west4-docker.pkg.dev/games-store-490014/games-store-repo/gamesplay-backend:v2
16 Image push succeeded Image is now stored in Artifact Registry and ready for Cloud Run deployment.

2. Cloud Run Deployment Workflow

Step Action Essential Commands / Notes
1 Deploy backend image to Cloud Run gcloud run deploy gamesplay-backend \ --image europe-west4-docker.pkg.dev/games-store-490014/games-store-repo/gamesplay-backend:v2 \ --platform managed \ --region europe-west4 \ --allow-unauthenticated
2 Open Cloud Run service settings Google Cloud Console → Cloud Run → gamesplay-backend → Edit & Deploy New Revision → Variables.
3 Add Django secret key SECRET_KEY=your-secret-key
4 Disable debug DEBUG=False
5 Configure database engine DB_ENGINE=django.db.backends.postgresql
6 Add Cloud SQL database name DB_NAME=your-db-name
7 Add Cloud SQL username DB_USER=your-db-user
8 Add Cloud SQL password DB_PASSWORD=your-db-password
9 Add Cloud SQL host DB_HOST=your-cloudsql-ip-or-connector
10 Add database port DB_PORT=5432
11 Add Cloudinary credentials CLOUDINARY_CLOUD_NAME=...
CLOUDINARY_API_KEY=...
CLOUDINARY_API_SECRET=...
12 Add CORS allowed origins CORS_ALLOWED_ORIGINS=https://your-frontend-domain.com
13 Save and deploy new revision Click Deploy in Cloud Run to apply variables and create a new revision.
14 Run migrations manually python manage.py migrate (locally or via a Cloud Run Job).
15 Verify Cloud Run logs Cloud Run → Logs → check for errors and successful startup.
16 Backend is live Use the URL provided by Cloud Run as your backend API endpoint.
